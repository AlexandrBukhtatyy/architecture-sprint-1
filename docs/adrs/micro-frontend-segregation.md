## Описание

### Проблема
Для перехода на микрофронтенды и удобства работы с ними необходимо реорганизовать проект.

### Предложение
Разделить кодовую базу по зонам ответственности, переиспользуемый код вынести в отдельные каталоги.

### Ограничения
Новая структура каталогов должна позволять:
* удобно и независимо вести разработку множеству команд
* иметь понятную структуру

### Аргументы
Переход на новую структуру каталогов даст следующие выгоды:
* Упростит понимание проекта
* Даст возможность разделить зоны ответственности между разными командами
* Упростит настройку CI/CD

### Реализация
Структура проекта
```
/frontend
  /public           // Файлы необходимые для сборки
  /src              // Исходники проекта
    /assets         // Файлы ассетов
      /css          // Общие стили приложения
      /svg          // Иконки
      /images       // Картинки
      /fonts        // Шрифты
    /libs           // Каталог с переиспользуемыми модулями, которые в полседствии можно публиковать в npm
      /api          // Функции взаимодействия с различными сервисами
      /ui           // Группирующий каталог
        /popup      // Компонент который можно переиспользовать в разных приложениях
      /modules      // Группирующий каталог
        /auth       // Модуль авторизации который можно переиспользовать в разных проектах
    /projects       // Группирующий каталог
      /host         // Хост-приложение
      /places       // микрофронтенд работы с галлереей локаций
      /profiles     // микрофронтенд профиля пользователя

```
Структура хост-приложения
```
/host
  /components       // Компоненты (header, footer и тд)
  /layouts          // layout - композиции базовых компонентов, формирующих страницу
  /utils            // Файлы с утилитами - обертки над чем то
  index.js

```
Структура микрофрентенда
```
/micro-frontend
  /components           // компоненты используемые только внутри этого микрофронтенда (следить что бы таких компонентов было как можно меньше)
  /features
    /feature
      feature.js        // компонент фичи
      feature.state.js  // хук с состянием компонента фичи
  /utils                // утилиты для упрохения кода в микрофронтенде (следить что бы таких утилит было как можно меньше)
```
