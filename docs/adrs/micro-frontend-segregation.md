## Описание

### Проблема
Для перехода на микрофронтенды и удобства работы с ними необходимо реорганизовать проект.

### Предложение
Разделить кодовую базу по зонам ответственности, переиспользуемый код вынести в отдельные каталоги.

### Ограничения
Новая структура каталогов должна позволять:
* удобно и независимо вести разработку множеству команд
* иметь понятную структуру

### Аргументы
Переход на новую структуру каталогов даст следующие выгоды:
* Упростит понимание проекта
* Даст возможность разделить зоны ответственности между разными командами
* Упростит настройку CI/CD
